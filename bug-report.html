#04 CRITICAL
Дублирующиеся HTML id — ai-messages-list, ai-input-field
assets/js/include.js · injectAiWidget() + injectUnifiedAiWidget()
▾
Оба виджета (injectAiWidget и injectUnifiedAiWidget) создают элементы с одинаковыми ID: ai-messages-list и ai-input-field. Браузер использует только первый найденный элемент при вызове document.getElementById() — второй виджет полностью сломан: сообщения не отправляются, история не заполняется.
Дублирование ID
// в injectAiWidget():
id="ai-messages-list"
id="ai-input-field"

// снова в injectUnifiedAiWidget():
id="ai-messages-list"   // коллизия!
id="ai-input-field"     // коллизия!
Решение
// injectAiWidget() — устаревший, не вызывается.
// Удалить его целиком ИЛИ переименовать ID:
id="ai-messages-list-legacy"
id="ai-input-field-legacy"
✓ Удалить мёртвый код injectAiWidget()
#05 CRITICAL
Двойная отправка сообщения (keydown + keypress)
assets/js/include.js · строки 498 и 504
▾
На поле ввода AI-чата навешено два обработчика нажатия Enter: один на keydown, другой на keypress. Во многих браузерах (Chrome, Edge) оба срабатывают — одно нажатие Enter отправляет сообщение дважды.
Баг — оба обработчика активны
inputField.addEventListener('keydown', (e) => {
  if (e.key === 'Enter' && !e.shiftKey) sendMessage();
});

inputField.addEventListener('keypress', (e) => {
  if (e.key === 'Enter' && !e.shiftKey) sendMessage();
});
Исправление — оставить только один
inputField.addEventListener('keydown', (e) => {
  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault();
    sendMessage();
  }
});
// keypress — удалить
✓ Удалить лишний обработчик keypress

#09 MAJOR

▾
Файл ai-widget.js создаёт резервный виджет-чат с захардкоженным русским текстом («Привет! Как помочь?», «Напишите сообщение...», «Отправить»). Если include.js не успеет загрузиться, пользователь увидит виджет на русском языке на турецком/английском сайте.

panel.innerHTML = `
  ...

  <input placeholder="Напишите сообщение...">

  ? { greeting: 'Hi! How can I help?', placeholder: 'Type a message...', send: 'Send' }
  : { greeting: 'Nasıl yardımcı olabilirim?', placeholder: 'Mesaj yazın...', send: 'Gönder' };
🟡 Незначительные
#10 MINOR
.btn-primary без background в базовом стиле
index.html · стиль .btn-primary
▾
Базовое правило .btn-primary задаёт только цвет текста и тени, но не задаёт background. Синий фон применяется через отдельный блок <style> в конце с флагом !important. Это хрупко — если переопределение не загрузится или будет переписано, кнопка будет прозрачной.
Баг — background задаётся только через override
.btn-primary {
  color: #020617;
  /* background — отсутствует! */
}
/* ... намного позже: */
.btn-primary { background: var(--accent) !important; }
Исправление — добавить в основное правило
.btn-primary {
  background: var(--accent);
  color: #020617;
}
#11 MINOR
3 отдельных блока <style> в <head> вместо одного
index.html · строки 25, 88, 80
▾
Стили главной страницы разбиты на три отдельных тега <style> в <head>, включая блок «Override» который перекрывает предыдущий блок. Это признак многократных несогласованных правок. Желательно объединить всё в один блок или вынести в site.css.
